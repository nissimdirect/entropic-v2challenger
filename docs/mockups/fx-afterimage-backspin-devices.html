Two effects, one id-vs-engine trick and one four-way stop-point switch — fx.afterimage (echo↔ghost combo) and fx.backspin (Decision 18's stop-point + ADSR platter curve).

Both device cards render through the existing generic ParamPanel → Knob/ParamChoice/ParamSlider path — no bespoke component is being proposed for either effect (confirmed zero Afterimage*.tsx / Backspin*.tsx in the repo). What's new here is which params are live at once: a `style` discriminator that greys an entire sibling param block (afterimage), and a `stop_mode` discriminator that swaps one inline control variant for another (backspin) — both DEPENDENT_PARAMS-gated, same grammar as `fx.copy_machine`'s `machine` switch.
