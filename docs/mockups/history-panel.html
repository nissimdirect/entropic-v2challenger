Design decisions & open questions

Locked in this mock: icon glyph set (✎⧉❄✦⌗) rendered verbatim from plan.md — not re-colored per op-class, since the classifier taxonomy (edit/routing/freeze/ paint/transform) is orthogonal to the routing-type palette (mask/field/scalar/feedback); inventing a color mapping between the two was not asked for. Open question 1: the task brief's example breadcrumb style was "Freeze C1 · ⌘Z", but P2's locked non-scope explicitly excludes "any change to the breadcrumb's text formatting beyond the raw description string." This mock renders the raw description only (no accelerator hint appended) to match the locked packet contract — flagging the discrepancy rather than silently picking one reading. Open question 2: child rows inside an expanded transaction render as plain indented text (matching plan.md's "nested read-only list") — whether children should also carry their own op-class icon is not specified anywhere in plan.md/packets.md; this mock leaves them icon-less as the more conservative reading. Open question 3: jump-progress cursor visual form (a filling track + moving dot under the footer) is this mock's best guess — plan.md only specifies "a lightweight in-progress indicator shown only when steps > 1," no exact form is locked.
