1Stage dot — one dot, two real meanings

§9.2, quoted verbatim in plan.md: pre = the source's decoded frame BEFORE its device chain. post = the source's frame AFTER apply_chain including its own mask stack — the track's finished contribution as it would enter the compositor. Under the original OD-1 default this dot was display-only (v1 only ever read pre). Under the T1 override it is a real, clickable toggle — flipping it changes what PK.2/PK.3's evaluator actually reads, not just a label.

A2 · drums-campre · luma
1click the chip to flip stage — hollow dot = pre (thumbnail = raw decode), filled dot = post (thumbnail visibly shifts, simulating "after A2's own device chain ran"). PK.8 — real stage TOGGLE replacing the old display-only dot.
why this matters: a tap on A2 in post mode sees A2 after its own invert/datamosh/whatever ran — genuinely different pixels than pre. This is why post needed its own packet (PK.3, VERY HIGH risk): resolving it means running part of A2's device chain early, in dependency order, sharing A2's own state slot so a stateful device never fires twice in one frame.

7Error state — source deleted, flat-0.5, never a crash

§9.1's trust-boundary rule, quoted verbatim in plan.md: missing track_id → flat-0.5 rule. Click "simulate: delete source" in the tray above — the chip flips to a red-dashed border, the thumbnail desaturates, and a rate-limited toast-shaped banner explains the degrade. This is a derived check (!tracks.some(t => t.id === node.params.track_id)), never a stored flag — undo restores instantly because there's no state to roll back, only a lookup that starts succeeding again.
