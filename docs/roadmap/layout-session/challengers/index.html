CREATRIX UX CHALLENGERS — Ableton × Photoshop

Three non-functional prototypes reconfiguring the same shipped component set (inventoried from entropic-v2challenger) along a blend spectrum — now life-size (real Live Signal token densities) with a STATES switcher in every prototype (bottom pill, or keys 1–8, Esc = default): BLEND · VOLUME · PERFORM · AUTOMATION · HISTORY · OPERATORS · EXPORT. Each state shows how that configuration absorbs the edge case in its own idiom — captions explain the move. Click a card to open; ⓘ annotations overlays the default-layout rationale. Challenger A also has a working SESSION ⇄ ARRANGE toggle.

Design system: Creatrix DS v1.2 draft (vocabulary · tokens · components · states · layout grammar) · icon directions (Hairline / Block / Interrupt — pick one)

B3 · Darkroom v3 — front-runner (user direction 2026-07-02)

B3 · DARKROOM v3 — the arrangement IS the layers

No layers panel anywhere. The bottom arrangement view is the layer stack (After Effects model): row order = z-order, and every track header carries eye · blend dropdown · opacity · ◨ per-track blending options · M/S. The twirl ▾ opens a track's effects and automation lanes inside the track itself. Right dock keeps only the effects browser + properties. Tool rail kept, grouped TRANSFORM / EDIT / MASK / MISC. All 8 states included. (B2's panel-based attempt is kept at challenger-b2-darkroom.html — rejected.)

A · SESSION
ABLETONPHOTOSHOP

Ableton-max. Clips become launchable session slots in track columns with per-track mixer strips (blend + opacity fader = channel strip). Scene column fires combos. Full-width bottom device chain. Preview docked right like clip detail.

  • Perform/pads system becomes a first-class view, not an overlay
  • SESSION ⇄ ARRANGE toggle (works in this mockup)
  • Photoshop DNA: only the mask-tool strip under preview
Risk: session slots imply clip-launch semantics the engine doesn't have yet (launch-quantized clip triggering exists only for pads).
C · SPLICE — the target blend
ABLETONPHOTOSHOP

The 50/50 hybrid. PS skeleton for space (tool rail + options bar + layers dock), Ableton skeleton for time (browser, arrangement timeline, full-width device chain, pads). One selection model: pick a layer OR a track OR a pad — chain, matrix and inspector all follow.

  • Layers panel ↔ timeline tracks: same objects, two views
  • Mod matrix docked under Layers — modulation always visible
  • Pad grid lives beside the device chain (perform without modal overlay)
Risk: densest of the three; needs the focus-mode (F) escape hatch to stay usable on a laptop.
B · DARKROOM
ABLETONPHOTOSHOP

Photoshop-max. Canvas is the hero (~70% of screen) with rulers + marching ants. Left tool rail unifies edit + mask tools; contextual options bar carries tool params. Tracks become a Layers panel (eye = mute, blend + opacity per layer, effects nested like layer styles). Timeline demoted to a filmstrip.

  • Effect browser as an "Adjustments" thumbnail grid
  • Properties panel = vertical device card for the selected effect
  • Ableton DNA: only transport + BPM survive up top
Risk: hides the time axis — automation lanes and multi-clip editing suffer; weakest fit for the "video DAW" identity (learning #239: market as Ableton-for-video, not Photoshop-for-video).
