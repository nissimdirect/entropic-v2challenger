0 · Vocabulary — the object model PROPOSED

The layer-tracks confusion came from naming, not structure. One name per object, everywhere — UI copy, docs, code comments. "Layer" is not an object; it's what a Track is when you think about z-order.

TermDefinitionWhere it livesNever call it
TrackA lane in the arrangement. Owns a device chain, opacity, blend mode, mute/solo. Row order = z-order (top = front).Arrangement view — its header IS the layer controllayer (as a separate panel/object)
ClipMedia on a track: in/out, speed, transform, per-clip opacity, matte stack refs.Track laneslot, region
DeviceOne effect instance in a chain (ON/OFF, A/B, mix, mask route, params).Device chain / twirl-down fx rowplugin, filter
ChainOrdered devices on a track or pad; terminal composite device = the track's blend/opacity.Device chain striprack (reserved for pad Rack)
OperatorModulation source (LFO, envelope, follower, Kentaro cluster…).Operators panel / routing canvasmodulator, LFO (generic)
MatteNode in a clip's mask stack (marquee/lasso/wand/key + ops).Mask stack, device mask routemask layer
PadPerformance trigger cell (gate/one-shot/toggle, key-bound, ADSR).Performance panel / Perform Roombutton
FieldImage/video assigned as a 2D per-pixel param source.Device param "Field…" routetexture map

1 · Color CANONICAL

Elevation = lightness, not shadows. Neutrals carry a cold hue-285 CRT cast. Two working accents (ACID, MOD) + three status colors — everything else is neutral. Category colors are 3px ticks ONLY, never fills or text.
